## 二、深度学习模型

### 2.1 EEGNet (eegnet)

紧凑型卷积神经网络，专为 EEG 信号设计（Lawhern et al., 2018）。

```bash
# 基础训练 (默认 100 epochs)
python model/run_pipeline.py --labels-csv labels.csv --model eegnet --verbose

# 自定义训练轮数
python model/run_pipeline.py --labels-csv labels.csv --model eegnet --epochs 200 --verbose
```

**特点：**
- 输入：2 通道 EEG 原始信号 (eeg_1_clean, eeg_2_clean)
- 窗口：500 点 (2 秒 @250Hz)
- 参数量：~2,000

### 2.2 Multimodal (multimodal)

时序多模态模型：生理特征序列 + 中文临床报告联合推理。

```bash
python model/run_pipeline.py --labels-csv labels.csv --model multimodal --epochs 80 --verbose
```

- 自动从数据库加载游戏特征 (game_hit_accuracy, game_score)

### 2.3 BIOT — 方案 A：自监督预训练 (patch-based)

基于 Patch token 化的 Transformer，在用户数据上做掩码信号建模预训练。

```bash
# 直接训练（无预训练）
python model/run_pipeline.py --labels-csv labels.csv --model biot --epochs 80 --verbose

# 自监督预训练 → 微调
python model/run_pipeline.py --labels-csv labels.csv --model biot --pretrain --epochs 80 --verbose
```

**流程：**
1. 预处理：50Hz 陷波 + 0.5-45Hz 带通滤波
2. 重采样：250Hz → 200Hz
3. 滑窗：4 秒窗口，2 秒重叠
4. Patch：每 50 点一个 patch，每通道 16 个 patch，共 64 个 token
5. 预训练：随机掩码 15% 的 patches，预测原始信号
6. 微调：添加分类头，端到端训练

**特点：**
- 输入：4 通道 (2 EEG + 2 EMG)
- 参数量：~1,221,890（信号模式）/ ~1,287,426（信号+眼动模式）
- 无需外部数据，适合小数据集

**眼动特征融合（混合融合）：**
BIOT 支持眼动特征融合（默认禁用），采用**混合融合策略**：
- **信号分支**：4 通道原始信号 → Patch 化 → Transformer 编码器 → 128 维 CLS 嵌入
- **眼动分支**：22 维眼动特征 → MLP 投影 → 64 维嵌入
- **融合**：拼接为 192 维向量 → 分类头

```bash
# BIOT 纯信号（默认）
python model/run_pipeline.py --labels-csv labels.csv --model biot --epochs 80 --verbose

# BIOT + 眼动
python model/run_pipeline.py --labels-csv labels.csv --model biot --eye-tracking --epochs 80 --verbose
```

> **注意**：当前数据集中 68% 的用户无眼动数据（填 0），眼动特征引入了噪声，BIOT 纯信号模式效果更优（见第六节对比）。

### 2.4 BIOT — 方案 B：官方预训练权重 (STFT-based)

使用官方 BIOT (NeurIPS 2023) 的预训练权重，基于 STFT 频谱 token 化。

#### 可用的预训练权重

| 权重文件 | 通道数 | 预训练数据 | 说明 |
|---------|--------|-----------|------|
| `EEG-PREST-16-channels.ckpt` | 16 | 5M 静息 EEG | MGH 脑电库 |
| `EEG-SHHS+PREST-18-channels.ckpt` | 18 | 5M+5M EEG | 静息 + 睡眠脑电 |
| `EEG-six-datasets-18-channels.ckpt` | 18 | 6 个数据集 | 推荐，泛化性最好 |

#### 冻结编码器 — 只训练分类头

适合数据量较少的场景，防止过拟合。只训练最后的分类头 (~2K 参数)，编码器保持预训练权重不变。

```bash
python model/run_pipeline.py --labels-csv labels.csv --model biot \
    --pretrain-ckpt model/pretrained/EEG-six-datasets-18-channels.ckpt \
    --freeze-encoder --epochs 80 --verbose
```

**训练参数：**
- 可训练参数：~514（分类头）
- 学习率：5e-4
- 早停 patience：15

#### 全模型微调 — 编码器 + 分类头

适合数据量充足的场景，对预训练权重进行端到端微调。

```bash
python model/run_pipeline.py --labels-csv labels.csv --model biot \
    --pretrain-ckpt model/pretrained/EEG-six-datasets-18-channels.ckpt \
    --epochs 80 --verbose
```

**训练参数：**
- 可训练参数：~3,183,622（全部）
- 学习率：5e-4（建议比冻结模式小，如 1e-4）
- 权重衰减：0.01
- 梯度裁剪：max_norm=1.0
- Label Smoothing：0.1

#### 不同预训练权重对比

```bash
# 使用 16 通道预训练权重
python model/run_pipeline.py --labels-csv labels.csv --model biot \
    --pretrain-ckpt model/pretrained/EEG-PREST-16-channels.ckpt \
    --freeze-encoder --epochs 80 --verbose

# 使用 18 通道预训练权重（推荐）
python model/run_pipeline.py --labels-csv labels.csv --model biot \
    --pretrain-ckpt model/pretrained/EEG-six-datasets-18-channels.ckpt \
    --freeze-encoder --epochs 80 --verbose
```

**输出文件：**
- `test/biot_model.pt` — 方案 A 模型
- `test/biot_official_model.pt` — 方案 B 模型
- `test/biot_pretrain.pt` — 自监督预训练权重（方案 A）
- `test/confusion_matrix.png` — 混淆矩阵
- `test/roc_curve.png` — ROC 曲线
- `result/{date}_{model}_n{samples}.txt` — 指标报告

### 2.5 Multimodal-BIOT — BIOT编码器 + 文本报告

用 BIOT 的 Patch-based Transformer 编码器替代多模态模型中的浅层 TemporalEncoder，直接从原始 EEG/EMG 信号中学习时序表示，并结合中文临床报告进行多模态联合推理。

```bash
# 直接训练（无预训练）
python model/run_pipeline.py --labels-csv labels.csv --model multimodal_biot --epochs 80 --verbose

# 自监督预训练 → 微调
python model/run_pipeline.py --labels-csv labels.csv --model multimodal_biot --pretrain --epochs 80 --verbose
```

**架构：**
```
原始信号 (B, 4, 800) → BIOTEncoder (PatchEmbed + TypeEmbed + 6层Transformer)
                     → sequence_out (B, 65, 128) + cls_out (B, 128)

中文报告 → ChineseBERT+LoRA → text_emb (B, 128)

CrossAttention: Q=text, KV=sequence → context (B, 128)

分类头: concat[cls_out(128), text_emb(128), context(128)] = 384 → 64 → 2
```

**特点：**
- 仅支持 CSV 标签模式（必须提供 `--labels-csv`）
- 信号路: 4 通道原始信号 (2 EEG + 2 EMG)，200Hz，4秒窗口
- 文本路: 从42维提取特征生成中文临床报告
- 用户级 train/test split，防止数据泄漏
- 可选自监督预训练 (`--pretrain`)

**输出文件：**
- `test/multimodal_biot_model.pt` — 训练好的模型
- `test/confusion_matrix.png` — 混淆矩阵
- `test/roc_curve.png` — ROC 曲线
- `result/{date}_multimodal_biot_n{samples}.txt` — 指标报告

```
用法: python model/run_pipeline.py [选项]

通用参数:
  --labels-csv PATH       诊断标签文件 (labels.csv)
  --model MODEL           模型选择: rf / xgb / lgb / eegnet / multimodal / biot / multimodal_biot

  --smote                 SMOTE 过采样 (仅 rf/xgb/lgb)
```

## 四、模型选择建议

| 场景 | 推荐模型 | 理由 |
|------|---------|------|
| 快速验证、基线对比 | `rf` | 训练快，可解释性强 |
| 特征工程已完善 | `xgb` / `lgb` | 梯度提升，精度高 |
| 端到端学习 | `eegnet` | 轻量 CNN，参数少 |
| 多模态融合 | `multimodal` | 结合游戏行为数据 |
| 原始信号 + 文本报告 | `multimodal_biot` | BIOT编码器 + 中文报告联合推理 |
| 数据量小 (<5000 样本) | `biot --pretrain-ckpt ... --freeze-encoder` | 预训练知识迁移 |
| 数据量中等 | `biot --pretrain-ckpt ...` | 全模型微调 |
| 无外部权重 | `biot --pretrain` | 自监督预训练 |

## 六、眼动特征对比实验

在 4765 个样本、90 个用户的数据集上进行对比实验，验证眼动特征对各模型的影响。

### 实验结果

| 模型 | Accuracy | F1 | AUC-ROC | 障碍类 F1 | 说明 |
|------|----------|------|---------|----------|------|
| **RF + 眼动** | **0.9238** | **0.9508** | **0.9823** | 0.83 | 眼动提升 +2.7% 准确率 |
| RF 纯信号 | 0.8967 | 0.9332 | 0.9572 | 0.77 | — |
| **BIOT 纯信号** | **0.9213** | **0.9444** | **0.9735** | **0.87** | 最佳深度学习结果 |
| BIOT + 眼动 | 0.9014 | 0.9291 | 0.9518 | 0.84 | 眼动引入噪声，略降 |

### 关键发现

1. **眼动特征对传统模型有显著帮助**：RF + 眼动的 AUC (0.9823) 是所有组合中最高的，眼动特征提供了有价值的补充信息
2. **眼动特征对 BIOT 有轻微负面影响**：当前数据集 68% 的用户（61/90）无眼动数据，填入的零向量作为噪声干扰了 BIOT 的端到端学习
3. **BIOT 纯信号表现优异**：AUC 0.9735，仅次于 RF + 眼动，说明 Transformer 架构能从原始信号中充分提取特征

### 推荐方案

| 场景 | 推荐配置 |
|------|---------|
| 默认（所有模型） | 不传 `--eye-tracking`，使用纯 EEG/EMG 信号 |
| 眼动覆盖率 >80% 的数据集 | `--eye-tracking`，预期有提升（尤其传统模型） |
